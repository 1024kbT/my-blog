# 把个人博客从展示页收成内容系统

这个项目一开始更像一个“酷炫模板首页”。视觉已经有了，但它离真正的博客还差关键几步：

- 文章没有独立页面
- 内容组织方式不稳定
- 更新文章需要直接改页面结构
- 评论和统计没有天然落点

## 为什么选 VitePress

我参考了朋友的博客后，基本确认目标不是 VuePress，而是 **VitePress**。

选它的理由很实际：

- 基于 Vite，构建快
- Markdown 写作体验直接
- 默认主题已经足够干净
- 很适合挂在 GitHub Pages 上

## 迁移后得到的东西

切过去之后，项目的重点终于从“首页怎么摆”变成了“内容怎么长”：

1. 首页只负责第一印象
2. 文章放进 `docs/posts`
3. 导航、归档、标签页都能独立存在
4. 评论和统计可以作为主题扩展插进去

## 这比从零手搓更值

如果你很确定自己后面会持续写内容，那从一开始就选更适合写作的结构，长期成本会低很多。
